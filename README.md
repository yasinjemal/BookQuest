# BookQuest 📖→🎮

A learning platform that turns any book or document (PDF, DOCX, Markdown, TXT)
into a bite-size, game-like course — Duolingo/Sololearn style, but the courses
are **generated by AI from any document**. Mobile-first PWA designed for
low-bandwidth and interrupted connections.
Built for learners everywhere, with African low-bandwidth contexts as the
design baseline.

## Platform features

- **Accounts** — email + password. The **first account registered becomes
  admin** and owns all pre-platform data.
- **Credits** — every new user gets 3 free course generations; generating a
  course costs 1 credit (admin is unlimited). Failed generations retry free.
- **Course library** — publish your generated courses with a category; anyone
  can explore, search, and enroll free.
- **Gamified learning** — XP, levels, daily streaks, weekly leaderboard, and
  spaced-repetition review of missed questions. All per-user.
- **Monetization** — credit packs & Premium subscription via **Flutterwave**
  (cards + mobile money across Africa). With no Flutterwave keys configured,
  billing runs in **test mode**: purchases succeed instantly without payment,
  so the whole flow is testable locally.
- **Marketplace-ready schema** — courses have `price_cents` and a
  `transactions` ledger, so paid courses + creator payouts can ship next
  without migrations.

## Setup

1. `npm install`
2. Put keys in `.env.local`:
   ```
   ANTHROPIC_API_KEY=sk-ant-...     # required for course generation
   FLW_SECRET_KEY=FLWSECK-...       # optional; omit for test-mode billing
   BILLING_CURRENCY=USD             # optional, default USD
   ```
3. `npm run dev` → http://localhost:3000
4. Register — your first account is the admin.

## Tech

- Next.js 15 (App Router) + TypeScript + Tailwind CSS 4
- Postgres (`pg`) — Neon in production; schema created idempotently in `lib/pg.ts`
- `@anthropic-ai/sdk` structured outputs (`zodOutputFormat` + `messages.parse`)
- Cookie sessions + bcryptjs password hashing (`lib/auth.ts`)
- Service worker: cached app shell; authenticated APIs stay network-only to
  prevent cross-account data leaks; answers queue safely during interruptions

## Key files

| File | Purpose |
|---|---|
| `lib/pg.ts` | Postgres pool + idempotent schema (Neon connection) |
| `lib/db.ts` | All queries (users, courses, progress, billing) |
| `lib/auth.ts` | Register/login, cookie sessions, route guards |
| `lib/billing.ts` | Products, Flutterwave checkout + verification, test mode |
| `lib/generator.ts` | Claude pipeline: outline call → per-module lesson calls |
| `lib/schemas.ts` | Zod schemas for cards/lessons/outline |
| `app/explore/page.tsx` | Public course library |
| `app/course/[id]/page.tsx` | Lesson path + owner publish controls |
| `app/profile/page.tsx` | Stats, leaderboard, store, account |
| `app/admin/page.tsx` | Admin dashboard (users, credits, revenue) |

## Going live (when ready)

1. Create a Neon Postgres database → set `DATABASE_URL` (the `-pooler` host)
   in the Vercel project's Environment Variables, plus `ANTHROPIC_API_KEY`
2. Create a Flutterwave merchant account → put `FLW_SECRET_KEY` in env
3. Deploy to Vercel — the schema is created automatically on first request
4. Point a domain, enable HTTPS — the PWA becomes installable for everyone
